Làm việc với GitHub: từ cá nhân đến teamwork

GitHub không chỉ là nơi lưu code. Đây là sân chơi trung tâm để lập trình viên quản lý phiên bản, trao đổi, review, theo dõi tiến độ, tự động kiểm thử và triển khai sản phẩm.

1. GitHub là gì?

Git

Công cụ quản lý phiên bản. Git ghi lại lịch sử thay đổi để bạn quay lại, so sánh và làm việc song song.

GitHub

Nền tảng lưu repository trên cloud, hỗ trợ Issues, Pull Requests, Projects, Actions và Wiki.

Branch

Một nhánh làm việc riêng để thử nghiệm mà không phá code chính.

Pull Request

Lời đề nghị nhập code vào nhánh chính, kèm review, comment và kiểm thử tự động.

2. Luồng làm việc nhóm

Issue
Branch
Commit
Pull Request
Merge

Hiểu nhanh: Git giống như máy ảnh chụp lịch sử code, còn GitHub là nơi cả nhóm cùng xem, góp ý, kiểm tra và triển khai.
